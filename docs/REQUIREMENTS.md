# Venduce 要件定義・設計方針

## プロジェクト概要

**Venduce**は「買ったモノを投稿して、つながる」をコンセプトにしたSNS × EC融合プラットフォームです。

ユーザーが購入した商品を投稿で紹介し、その投稿経由での購入が追跡されます。フォロワーへの購買影響力がバッジ・ランキングとして可視化されることで、購買インフルエンサーとしてのモチベーションが生まれる仕組みを目指しました。

---

## システムアーキテクチャ

| レイヤー | 技術選定 | 選定理由 |
|---------|---------|---------|
| バックエンド | FastAPI + SQLAlchemy + Alembic + PostgreSQL | 型安全なAPI開発、高パフォーマンス、マイグレーション管理の容易さ |
| フロントエンド | Next.js 15 + React 19 + TypeScript + Tailwind | App Routerによるサーバーコンポーネント活用、型安全 |
| 認証 | JWT (RS256 / HS256) | 非対称鍵で署名・検証の責務分離 |
| ID生成 | ULID | ソート可能でURLセーフ、UUID代替 |
| ファイル保管 | ローカルストレージ | 開発・デモ環境での簡易性 |
| 開発環境 | Docker Compose + Nginx リバースプロキシ | 環境差異を排除し、本番構成に近い開発体験 |
| リアルタイム | WebSocket | 通知のリアルタイム配信基盤 |

## バックエンド設計方針

### 2層アーキテクチャ（Router / Service）

最初から層を増やしすぎず、必要になったタイミングで分割する方針を取っています。

| 層 | 責務 | 禁止事項 |
|---|------|---------|
| **Router** | リクエスト/レスポンス変換、認可チェック、Serviceの呼び出し | ビジネスロジックの記述 |
| **Service** | ユースケース単位のビジネスロジック、トランザクション境界の管理、DBアクセス | — |

クエリが複雑化・重複してきた場合はRepositoryを切り出して集約します。過度な事前設計よりも、実際のコード規模に合わせて段階的にモジュールを分割する方針です。

### スキーマ設計

- `UserCreate` / `UserUpdate` / `UserRead` のように用途別にPydanticスキーマを分割
- `password_hash` などの内部フィールドはResponseスキーマに含めない
- ドメイン共通のEnum値は `app/models/enums.py` に集約し、文字列ハードコードを禁止

### 認証設計

- アクセストークン（RS256）＋ リフレッシュトークン（DB管理）の二段構成
- メール確認フロー（`is_confirmed` フラグ）
- 管理者権限（`is_admin` フラグ）

## 実装済み機能一覧

### ユーザー管理

- ユーザー登録・ログイン・ログアウト
- JWT認証 + リフレッシュトークン
- メール確認フロー
- プロフィール取得・更新・画像アップロード
- ユーザー検索
- 購入履歴の公開/非公開設定

### 投稿・SNS

- 投稿作成・編集・削除（キャプション・複数画像・商品リンク・タグ）
- 投稿ステータス管理（public / draft / archived）
- ビュー数・いいね数・購入数の自動カウント
- いいね追加・削除
- コメント投稿・編集・削除（ネスト返信対応）
- フォロー・アンフォロー
- フォロー中フィード

### 商品管理

- 商品CRUD（SKU・価格・在庫・ステータス）
- カテゴリ管理（多対多）
- ブランド管理
- 商品画像アップロード（複数）
- メタデータ保存（JSONB）

### 購入・支払い

- 購入登録API
- 購入と投稿の帰属記録（`referring_post_id` による購買影響追跡）
- 購入ステータス管理
- 購入履歴取得
- 支払い方法登録・管理

### ゲーミフィケーション

- バッジシステム（Silver / Gold / Platinum）
- バッジ自動付与ロジック（購買影響力に基づく）
- バッジ獲得時のアニメーション

### タグ・検索

- タグ定義・管理
- 投稿へのタグ付与
- タグ検索・フィルタリング

### 通知

- 通知記録（Like / Follow / Comment / Purchase / Ranking / Badge）
- 通知一覧取得・既読フラグ管理
- WebSocket基盤（リアルタイム通知に対応済み）

### フロントエンド画面

- ホーム・ログイン・会員登録
- フィード・投稿作成・投稿詳細
- 商品一覧・商品詳細
- プロフィール・プロフィール設定
- 購入確認・購入完了・購入履歴
- 注文管理・通知一覧
- ユーザー検索・全体検索
- SQLAdmin管理画面（ユーザー・商品・カテゴリ・ブランド）
